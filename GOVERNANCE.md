# Governance

## Project Owner

Smart! is developed and maintained by **Finalomega**. The maintainer has final authority over all aspects of the project, including:

- Roadmap and feature direction
- Architecture and design decisions
- Pull request acceptance or rejection
- Release timing and versioning
- License terms
- Community standards and moderation

## Decision Making

Day-to-day decisions are made by the maintainer directly. Community input is welcomed through GitHub Issues, pull requests, and the Discord server, but suggestions do not carry a vote or guarantee of implementation.

A pull request may be declined for any reason, including reasons of project direction, scope, or timing. The maintainer will provide an explanation in all cases.

## Releases

Official releases of Smart! are published exclusively by the maintainer:

- Via [ficsit.app/mod/SmartFoundations](https://ficsit.app/mod/SmartFoundations)
- Via the official GitHub repository under the Finalomega account

No other release source is authorized. See [TRADEMARKS.md](TRADEMARKS.md) for branding rules that apply to forks.

## Contribution Acceptance

Contributions are accepted at the maintainer's discretion. See [CONTRIBUTING.md](CONTRIBUTING.md) for the process and contribution license terms.

## Security and Stability

Security issues and crashes that could affect save data are prioritized over feature work. See [SECURITY.md](SECURITY.md) for reporting instructions.

## Changes to This Document

The maintainer may update these governance terms at any time. The current version in the main branch is authoritative.
